1 · Executive Summary

UP Labs = แพลตฟอร์มสุขภาพ-longevity ของ UP Wellness (Next.js 14 App Router + Supabase + Vercel). v1 ถูกติว่า "ไม่สวย/ใช้ยาก" เพราะ UI ไม่ตรง brand ตัวเอง (glass/aurora/emoji/mono-uppercase). v2 = redesign "clinical-warm" สร้างขนานใต้ /v2/* โดยใช้ API + data เดิม แล้วค่อย cutover.

5 · Design System (clinical-warm — implemented)

  • สี token เดิม (rose/wellness/science/amber/ink/surface) · status text สีเข้ม AA (caution #854D0E · good #3F6212 · optimal #15803D · warning #9A3412 · danger #991B1B) สำหรับตัวอักษร · bright hex เฉพาะ dot/ring
  • เลิก glass/aurora/emoji-icon/mono-uppercase → การ์ดขาว rounded-2xl + เงานุ่ม · Lucide icons · Sarabun sentence-case · text ≥12px · touch ≥44px
  • macro C:P:F สีเดียวทั้งแอป: คาร์บ=rose · โปรตีน=wellness · ไขมัน=amber (NutriScan + Plate + CPFPie default) · CPFPie รับ prop colors ได้
  • Report CI แยก (deliverable): olive #3D5826 / gold / cream — ใช้กับ Wearable Report (Pook), Med-Map, Lab Report
  • empty/loading/error states ครบ · app/v2/error.tsx error boundary

6 · Global Patterns (implemented)

  • App Shell ถาวรทุกหน้า: โลโก้ + app-switcher (ชี้ /v2 ทุกแอปที่มี) + breadcrumb + user/role indicator (Shell fetch /api/debug/me เองเมื่อหน้าไม่ส่ง prop)
  • breadcrumb ทุกหน้า detail (+ mobile row)
  • Report มี TOC index sticky scroll-jump (โชว์เฉพาะ section ที่มีข้อมูล)
  • responsive 375px+ · ตาราง→card/column-shed บนมือถือ

11 · Changelog & Gotchas (UAT lessons)

Runtime bugs ที่ static review/build ไม่จับ — แก้แล้วทั้งหมด:

อาการสาเหตุ + แก้
หน้า customer/pulse detail erroruse(params) (Next 15 pattern) บน Next 14 → params เป็น object ไม่ใช่ Promise → React.use() throw · แก้: sync const {id}=params
ชื่อผู้ใช้ไม่ขึ้นในหน้าอื่นShell รับ profile เป็น prop แต่หน้า client ไม่ส่ง · แก้: Shell fetch /api/debug/me เอง
/v2/healthcheck crashmetaflex lead เก็บ risk_level="WARNING ZONE ⚠️" · RISK_META[x] undefined · แก้: defensive riskMeta() (+ prospects tier/status เชิงรุก)
back กลับ v1Shell app-switcher ชี้ v1 · แก้: V2_ROUTE map
